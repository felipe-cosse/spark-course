# Exercises: Glossary and Concept Connections

Source: [Course glossary](../../reference/glossary.md)

Estimated time: 45–75 minutes. Difficulty: mixed.

## Key terms reinforced

| Term | Definition |
|---|---|
| Concept map | Diagram connecting terms through labeled relationships. |
| Retrieval practice | Recall performed without reading the answer first. |
| Contrast | Explaining how related terms differ. |
| Example | Concrete instance illustrating an abstract concept. |
| Misconception | Plausible but incorrect understanding. |

## Exercise 1: Execution concept map

Connect action, job, stage, task, partition, shuffle, driver, and executor. Every arrow must have a verb phrase such as “triggers,” “contains,” or “runs on.”

## Exercise 2: Contrast pairs

Explain each pair in two sentences and one example:

- transformation versus action;
- storage partition versus execution partition;
- event time versus processing time;
- DataFrame versus RDD;
- checkpoint versus cache;
- idempotency versus exactly-once claim.

## Exercise 3: Correct misconceptions

Refute: “one file equals one partition,” “more partitions are always faster,” “watermark drops all old data,” “broadcast means no memory risk,” and “a successful retry cannot duplicate output.”

## Self-check

- Do definitions explain behavior, not repeat the term?
- Does every abstract term have an example?
- Are related terms connected into a model?
- Can you recognize common incorrect explanations?

## Stretch task

Teach the execution concept map aloud in under three minutes and ask a listener to reconstruct it without seeing yours.
