# PySpark Course Exercise Index

This directory contains one exercise companion for every course page. Complete the lesson first, then attempt its exercises without copying the lesson example. Use hints only after writing down your first approach.

## Key terms on this page

| Term | Definition |
|---|---|
| Exercise | A task designed to practice or assess a specific skill. |
| Deliverable | The concrete artifact submitted as evidence of completed work. |
| Hint | Limited guidance that helps unblock reasoning without supplying a full solution. |
| Acceptance criteria | Observable conditions the deliverable must satisfy. |
| Stretch task | Optional work that introduces additional complexity or independence. |
| Evidence | Code, output, tests, plans, or written reasoning that supports a claim. |

## How to use the exercises

For code exercises, submit the code plus a short note that states input grain, output grain, actions, and likely shuffle boundaries. For design exercises, state assumptions and distinguish guarantees supplied by Spark from those supplied by storage, the catalog, the orchestrator, or another platform.

Recommended evidence bundle:

- source code or query;
- tiny explicit-schema fixture;
- test or expected result;
- `explain(mode="formatted")` output when relevant;
- answers to the self-check questions;
- one limitation or trade-off.
