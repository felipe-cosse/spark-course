# Exercises: Rubric and Engineering Review

Source: [Capstone rubric and review](../../07-capstones/03-rubric-and-review.md)

Estimated time: 90–150 minutes. Difficulty: advanced review practice.

## Key terms reinforced

| Term | Definition |
|---|---|
| Rubric | Observable scoring guide across quality dimensions. |
| Finding | Evidence-linked statement of risk or gap. |
| Severity | Urgency/impact category for a finding. |
| Verification | Evidence demonstrating remediation closed the risk. |
| Known limitation | Explicitly documented unsolved constraint. |

## Exercise 1: Calibrate scoring

Score three hypothetical submissions:

- polished demo with no retry evidence;
- correct code with no security/runbook artifacts;
- plain presentation with tests, recovery drill, and reconciliation.

Defend every dimension score with evidence and identify missing evidence rather than guessing.

## Exercise 2: Write findings

Convert vague comments into evidence–risk–recommendation–verification findings:

- “joins look wrong”;
- “needs more tests”;
- “streaming is unsafe”;
- “job is expensive”;
- “security should improve.”

## Exercise 3: Peer review

Review a capstone or course exercise using the checklist. Separate blocking findings, improvements, questions, and known limitations. Allow the author to challenge findings with new evidence.

## Self-check

- Are scores reproducible by another reviewer?
- Does severity reflect impact and likelihood?
- Is each recommendation the smallest verifiable change?
- Does verification test the risk rather than only code presence?

## Stretch task

Run a second review after remediation and write a closure note for each finding, including evidence or reason it remains open.
